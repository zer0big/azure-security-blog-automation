# 배포 가이드

## 사전 요구사항

### 필수 도구

1. **Azure CLI** (버전 2.50.0 이상)
   - Windows: `winget install Microsoft.AzureCLI`
   - Mac: `brew install azure-cli`
   - Linux: [설치 가이드](https://learn.microsoft.com/ko-kr/cli/azure/install-azure-cli-linux)

2. **Azure Functions Core Tools** (버전 4.x)
   - Windows: `npm install -g azure-functions-core-tools@4`
   - Mac: `brew install azure-functions-core-tools@4`
   - Linux: [설치 가이드](https://learn.microsoft.com/ko-kr/azure/azure-functions/functions-run-local)

3. **.NET 8 SDK**
   - 다운로드: https://dotnet.microsoft.com/download/dotnet/8.0
   - 확인: `dotnet --version` (8.0.x 이어야 함)

4. **Git**
   - 다운로드: https://git-scm.com/downloads
   - 확인: `git --version`

5. **(선택 사항) Visual Studio Code**
   - 다운로드: https://code.visualstudio.com/
   - 권장 확장:
     - Azure Functions
     - Azure Logic Apps (Standard)
     - Bicep

### Azure 리소스 사전 요구사항

1. **활성 Azure 구독**
   - 액세스 확인: `az account show`
   - 구독 설정: `az account set --subscription <subscription-id>`

2. **Azure OpenAI Service**
   - **모델**: GPT-4o 배포
   - **지역**: 지원되는 모든 지역 (Logic App과 동일한 지역 권장, 낮은 지연 시간)
   - **참고**: Azure OpenAI 액세스 권한이 없는 경우 요청 필요
   - **문서**: [Azure OpenAI 빠른 시작](https://learn.microsoft.com/ko-kr/azure/ai-services/openai/quickstart)

3. **필수 Azure 권한**
   - 구독 또는 리소스 그룹에 대한 `기여자` 역할
   - 다음을 생성할 수 있는 권한:
     - Storage Accounts
     - App Service Plans
     - Function Apps
     - Logic Apps (Standard)
     - Application Insights

4. **Office 365 계정**
   - 이메일 알림 발송에 필요
   - Logic Apps에서 Office 365 Connector를 사용할 수 있는 권한 필요

## 단계별 배포

### 1단계: 리포지토리 클론

```bash
git clone https://github.com/zer0big/azure-security-blog-automation.git
cd azure-security-blog-automation
```

### 2단계: Azure OpenAI 구성

#### 옵션 A: 기존 Azure OpenAI 사용

1. Azure OpenAI 엔드포인트 및 키 가져오기:
   ```bash
   # Azure OpenAI 리소스 목록 조회
   az cognitiveservices account list --query "[?kind=='OpenAI'].{Name:name, Endpoint:properties.endpoint, ResourceGroup:resourceGroup}" --output table
   
   # API 키 가져오기
   az cognitiveservices account keys list --name <your-openai-resource-name> --resource-group <resource-group> --query key1 --output tsv
   ```

2. 환경 변수 설정:
   ```powershell
   # PowerShell
   $env:AZURE_OPENAI_ENDPOINT = "https://<your-openai-resource>.openai.azure.com/"
   $env:AZURE_OPENAI_KEY = "<your-api-key>"
   ```
   
   ```bash
   # Bash
   export AZURE_OPENAI_ENDPOINT="https://<your-openai-resource>.openai.azure.com/"
   export AZURE_OPENAI_KEY="<your-api-key>"
   ```

#### 옵션 B: 새 Azure OpenAI 리소스 생성

```bash
# Azure OpenAI 리소스 생성
az cognitiveservices account create \
  --name "aoai-security-blog-automation" \
  --resource-group "rg-security-blog-automation-dev" \
  --location "koreacentral" \
  --kind "OpenAI" \
  --sku "S0"

# GPT-4o 모델 배포
az cognitiveservices account deployment create \
  --name "aoai-security-blog-automation" \
  --resource-group "rg-security-blog-automation-dev" \
  --deployment-name "gpt-4o" \
  --model-name "gpt-4o" \
  --model-version "2024-08-06" \
  --model-format "OpenAI" \
  --sku-capacity 10 \
  --sku-name "Standard"
```

인프라 디렉토리로 이동:

```bash

```

```powershell
.\deploy.ps1 `
  -AzureOpenAIEndpoint $env:AZURE_OPENAI_ENDPOINT `
  -AzureOpenAIKey $env:AZURE_OPENAI_KEY `
  -ResourceGroupName "rg-security-blog-automation-dev" `
  -Location "koreacentral"
```

#### 옵션 B: Bash로 배포 (Linux/Mac)

```bash
chmod +x deploy.sh

./deploy.sh \
  --openai-endpoint "$AZURE_OPENAI_ENDPOINT" \
  --openai-key "$AZURE_OPENAI_KEY" \
  --resource-group "rg-security-blog-automation-dev" \
  --location "koreacentral"
```

#### API 연결 구성

워크플로우는 인증이 필요한 3가지 유형의 연결을 사용합니다:

##### 1. Office 365 연결 (이메일 발송용)

1. Logic App Designer에서 "Send_an_email_(V2)" 액션 열기
2. "Change connection" 클릭
3. "+ Add new" 클릭
4. Office 365 계정으로 로그인
5. 이메일 발송 권한 부여
6. 워크플로우 저장

##### 2. RSS 연결 (피드 가져오기용)

워크플로우에는 5개의 RSS 액션이 있습니다. 이들은 기본 제공 커넥터이며 일반적으로 추가 구성이 필요하지 않습니다.

연결 오류가 발생하는 경우:
1. Designer에서 각 RSS 액션 열기
2. 피드 URL이 올바른지 확인
3. 연결 테스트

##### 3. HTTP 액션 (Azure Functions 호출용)

배포된 Function App을 가리키도록 HTTP 액션 URL을 업데이트합니다:

1. "SummarizePost_HTTP" 액션 열기
2. URI 업데이트: `https://func-dev-security-blog-automation.azurewebsites.net/api/SummarizePost?code=<function-key>`
3. "GenerateEmailHtml_HTTP" 액션 열기
4. URI 업데이트: `https://func-dev-security-blog-automation.azurewebsites.net/api/GenerateEmailHtml?code=<function-key>`

**Function Key 가져오기:**
```bash
az functionapp function keys list \
  --name func-dev-security-blog-automation \
  --resource-group rg-security-blog-automation-dev \
  --function-name SummarizePost \
  --query default --output tsv
```

## 문제 해결

### 일반적인 문제

#### 1. Bicep 배포 실패

**오류**: `Resource 'Microsoft.Storage/storageAccounts' already exists`

**해결책**:
```bash
# 기존 리소스 그룹 삭제
az group delete --name rg-security-blog-automation-dev --yes

# 재배포
cd infra
.\deploy.ps1 -AzureOpenAIEndpoint $env:AZURE_OPENAI_ENDPOINT -AzureOpenAIKey $env:AZURE_OPENAI_KEY
```

#### 2. Function App 응답 없음

**오류**: `503 Service Unavailable`

**해결책**:
```bash
# Function App 재시작
az functionapp restart --name func-dev-security-blog-automation --resource-group rg-security-blog-automation-dev

# 로그 확인
az functionapp log tail --name func-dev-security-blog-automation --resource-group rg-security-blog-automation-dev
```

#### 3. Logic App 연결 오류

**오류**: `Unauthorized` 또는 `Connection not authenticated`

**해결책**:
1. Logic App → API 연결로 이동
2. 각 연결 클릭 (Office 365, RSS)
3. 연결 재인증
4. 워크플로우 다시 테스트

#### 4. Azure OpenAI 속도 제한

**오류**: `429 Too Many Requests`

**해결책**:
```bash
# Azure OpenAI 할당량 증가
az cognitiveservices account deployment update \
  --name <your-openai-resource> \
  --resource-group <resource-group> \
  --deployment-name gpt-4o \
  --sku-capacity 20
```

#### 5. 이메일 수신 안 됨

**체크리스트**:
- [ ] Office 365 연결 인증됨
- [ ] "Send_an_email_(V2)" 액션의 이메일 수신자 주소 정확함
- [ ] Logic App 실행 기록에 성공 표시
- [ ] 스팸/정크 폴더 확인
- [ ] Office 365 계정에 이메일 발송 권한 있음

#### 6. Table Storage 액세스 거부

**오류**: `AuthorizationFailure` 또는 `403 Forbidden`

**해결책**:
```bash
# Function App 관리 ID에 Storage Table Data Contributor 역할 할당
FUNCTION_PRINCIPAL_ID=$(az functionapp identity show --name func-dev-security-blog-automation --resource-group rg-security-blog-automation-dev --query principalId --output tsv)

az role assignment create \
  --assignee $FUNCTION_PRINCIPAL_ID \
  --role "Storage Table Data Contributor" \
  --scope "/subscriptions/<subscription-id>/resourceGroups/rg-security-blog-automation-dev/providers/Microsoft.Storage/storageAccounts/stdevsecurityblog"
```

   - **피드 이름**: 이모지 + 설명 이름
4. 동일한 처리 흐름에 연결
5. 저장
